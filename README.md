# mle-template-case-sprint2

Здесь укажите имя вашего бакета: s3-student-mle-20251124-4e76e7f4c1

ML-проект для прогнозирования стоимости недвижимостию Наша цель — сделать процесс воспроизводимым и улучшить ключевые модельные метрики, которые влияют на бизнес-показатели компании, в частности, на увеличение количества успешных сделок.

Чтобы запустить проект нужны следующие шаги:

    git clone https://github.com/akzholtech/mle-project-sprint-2.git

    cd mle-project-sprint-2

    pip install -r requirements.txt 

В ходе разработки были использованы инструменты такие как:

    - Mlflow - для отслеживние и востроизводимости модели

    - база данных PostgreSQL с данными для обучения

    - база данных PostgreSQL с данными для MLflow

    - объектное хранилище для MLflow


Этап 1: Разворачивание MLflow и регистрация модели

Для воспроизведения процесса развертывания MLflow необходимо выполнить следующие команды:

    Запустить виртуальную среду:
    source .venv/bin/activate

    Загрузить переменные окружения:
    export $(cat .env | xargs)

    Запустить MLflow сервер:
    sh mlflow_server/run_mlflow_server.sh

    Используемый эксперимент:
    experiment_id = 11
    experiment_name = Prev_linear_model

Этап 2: Проведение EDA

    В датасет был добавлен новый признак — цена за квадратный метр.

    Максимальная цена недвижимости составляет примерно 30 млн.

    Минимальная цена — около 1 млн.

    Целевая переменная линейно зависит от общей площади.

Этап 3. Генерация признаков и обучение модели

    На этом этапе были использованы методы генерации признаков: PolynomialFeatures, KBinsDiscretizer и AutoFeatRegressor

    Результаты модели были оценены с помощью метрик r2_score и mean_absolute_error.

Этап 4. Отбор признаков и обучение новой версии модели

    Для отбора наиболее важных признаков применялись методы backward feature selection и forward feature selection

    В результате были получены наиболее оптимальные значения метрик модели (~0.95).

Этап 5. Подбор гиперпараметров и обучение новой версии модели

    На последнем этапе был выполнен подбор гиперпараметров. Поскольку у Linear Regression практически нет гиперпараметров, оптимизация проводилась для параметров fit_intercept и positive с использованием библиотеки Optuna и метода GridSearchCV

    В результате были получены практически идентичные значения метрик модели.
